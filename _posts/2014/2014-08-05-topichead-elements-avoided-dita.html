I know this may sound like a trivial topic, but I wanted to note that the topichead element used in ditamap files should mostly be avoided. The topichead element allows you to group topics into a hierarchy, like this:

[xml]
<topichead navtitle="Links">
<topicref href="keyref_links.dita"/>
<topicref href="inline_links.dita"/>
<topicref href="related_links.dita"/>
<topicref href="relationship_tables.dita"/>
</topichead>
[/xml]

The problem is that the topichead element output creates an empty folder. When the user clicks the folder, no content appears. The folder just expands to show its inner contents. Given that most other elements clicked in the TOC change what appears in the main content window, having an empty folder not show any change may cause users to assume something's broken.

Beyond the usability issue, it's nice to have a mini-TOC as the opening page for each folder. When you use parent-child relationships with the topicref elements in a map, the child topics appear embedded on the parent topic along with the shortdesc element for each topic.

For example, suppose this is your structure:

[xml]
<topicref href="toc_links.dita">
<topicref href="keyref_links.dita"/>
<topicref href="inline_links.dita"/>
<topicref href="related_links.dita"/>
<topicref href="relationship_tables.dita"/>
</topicref>
[/xml]
