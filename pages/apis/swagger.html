Swagger provides a specification that various tools can use to generate REST API doc in an interactive UI interface. Swagger is described as both a specification and a framework, which is somewhat vague. It's vague because Swagger can be implemented in a variety of ways and within a lot of different platforms and languages. You can incorporate this syntax in your actual code (particular to a specific language). Or you can create a YAML file that follows a particular syntax that the Swagger-UI parser can process.

With the latter method, it's kind of like DITA, in that you have a syntax that you format content with, and then various tools can parse that syntax and render it into an output.

There's an online editor you can use to create a Swagger YAML file in the correct syntax. You can use the Swagger editor at http://editor.swagger.io/#/edit. You can also run this editor locally by following the instructions here. Here's the Swagger editor project page.
